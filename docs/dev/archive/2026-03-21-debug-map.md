# Debug Map Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a feature-gated live debug map that visualises prefetch state in a browser — aircraft position, sliding box bounds, DSF region states, and pipeline metrics — enabling real-time diagnosis of prefetch coverage gaps.

**Architecture:** An `axum` HTTP server (gated behind `debug-map` Cargo feature) starts alongside the main service. It serves a single HTML page with a Leaflet.js map that polls a JSON API endpoint every 2 seconds. The API collects state from existing thread-safe accessors (`SharedAircraftPosition`, `GeoIndex`, `SharedSimState`, `SharedPrefetchStatus`) without adding any new shared state. The Leaflet map renders DSF regions as colour-coded rectangles and the prefetch box as a translucent overlay.

**Tech Stack:** `axum` 0.7 (feature-gated), `serde_json` (existing), Leaflet.js via CDN (no bundling), existing `tokio` runtime.

---

## Feature Gate Design

All debug map code is behind `#[cfg(feature = "debug-map")]`. Standard builds are unaffected:

```toml
# Cargo.toml
[features]
debug-map = ["dep:axum"]

[dependencies]
axum = { version = "0.7", optional = true }
```

```bash
# Build with debug map
cargo build --features debug-map

# Standard build (no debug map)
cargo build
```

The server auto-starts when the feature is compiled in. No CLI flag needed — if you built with `debug-map`, you want it.

## Task 3: Axum HTTP server

**Files:**
- Create: `xearthlayer/src/debug_map/server.rs`
- Modify: `xearthlayer/src/debug_map/mod.rs`

Minimal axum server with two routes: `GET /` (HTML map page) and `GET /api/state` (JSON).

- [ ] **Step 1: Implement `DebugMapServer`**

```rust
use axum::{Router, Json, response::Html, extract::State};
use std::net::SocketAddr;
use tokio_util::sync::CancellationToken;

use super::api::{collect_snapshot, DebugStateSnapshot};
use super::html::MAP_HTML;
use super::state::DebugMapState;

pub struct DebugMapServer {
    state: DebugMapState,
    port: u16,
}

impl DebugMapServer {
    pub fn new(state: DebugMapState, port: u16) -> Self {
        Self { state, port }
    }

    /// Start the server. Runs until cancellation.
    pub async fn run(self, cancellation: CancellationToken) {
        let app = Router::new()
            .route("/", axum::routing::get(serve_map))
            .route("/api/state", axum::routing::get(serve_state))
            .with_state(self.state);

        let addr = SocketAddr::from(([0, 0, 0, 0], self.port));
        tracing::info!(port = self.port, "Debug map server started at http://localhost:{}", self.port);

        let listener = tokio::net::TcpListener::bind(addr).await.unwrap();
        axum::serve(listener, app)
            .with_graceful_shutdown(cancellation.cancelled_owned())
            .await
            .unwrap();
    }
}

async fn serve_map() -> Html<&'static str> {
    Html(MAP_HTML)
}

async fn serve_state(State(state): State<DebugMapState>) -> Json<DebugStateSnapshot> {
    Json(collect_snapshot(&state))
}
```

## Task 4: Leaflet.js map page

**Files:**
- Create: `xearthlayer/src/debug_map/html.rs`

Single embedded HTML string with Leaflet.js map. No external files — everything is inline or loaded from CDN.

- [ ] **Step 1: Implement `MAP_HTML` constant**

The HTML page:
- Loads Leaflet.js from CDN
- Renders fullscreen map centered on aircraft (or world if no position)
- Polls `/api/state` every 2 seconds
- Draws:
  - **Aircraft marker** — blue dot with heading arrow
  - **Prefetch box** — translucent blue rectangle
  - **DSF region grid** — coloured rectangles:
    - Green (#4CAF50) = Prefetched
    - Yellow (#FFC107) = InProgress
    - Grey (#9E9E9E) = NoCoverage
    - Purple (#9C27B0) = Patched
    - Blue outline = Retained
  - **Stats overlay** — top-right panel with cache hit rate, mode, sim state

Key implementation details:
- Use `L.rectangle()` for each DSF region (1°×1° bounds)
- Use `L.rectangle()` for prefetch box with dashed border
- Use `L.circleMarker()` for aircraft position
- Auto-pan to follow aircraft
- Clear and redraw regions each poll (simple, no diffing)

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| `axum` dependency too heavy | Optional dep, not in standard builds. axum is thin over hyper (already transitively present) |
| Debug server port conflicts | Default 8087, configurable. Different from X-Plane Web API (8086) |
| Performance impact from polling | 2-second poll interval, GeoIndex reads are lock-free (DashMap shards). No measurable impact |
| CDN dependency for Leaflet.js | Could embed Leaflet locally later, but CDN is simpler for a debug tool |
| HTML/JS quality | This is a debug tool, not user-facing. Functional > pretty |
| Standard build regression | Feature gate ensures zero impact. CI runs without `debug-map` flag |
